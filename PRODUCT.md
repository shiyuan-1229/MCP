# Product

## Register

product

## Users

MCP Forge is used by delivery consultants, platform operators, and customer-side project owners. Operators use it to track customer projects, package MCP capabilities, run sandbox calls, publish versions, manage gateway policy, review usage, and prepare billing or delivery files. Customers use it to inspect delivered MCP assets, call quality, effect data, and handover materials.

## Product Purpose

The product is an enterprise MCP delivery and governance platform. It turns existing business systems into governed MCP assets, tracks the lifecycle from data source access to testing and release, and keeps usage, gateway, billing, and deliverable evidence in one admin surface. Retail data is a demo source only, not the formal product boundary.

## Brand Personality

Calm, precise, trustworthy. The interface should feel like a serious delivery control center: clear enough for customers, dense enough for operators, and restrained enough for long daily use.

## Anti-references

Do not make the product look like a retail ERP, inventory dashboard, or sales analysis system. Avoid marketing-style hero pages, decorative card grids, oversized metrics without workflow context, and demo retail modules presented as core navigation.

## Design Principles

- Keep the information architecture centered on the MCP lifecycle: project, asset, release, gateway, usage, billing, deliverables.
- Separate platform work from demo data sources in both navigation and copy.
- Show operational evidence before decoration: statuses, owners, versions, trace IDs, release windows, and delivery files.
- Make role boundaries obvious: admins can see all customers and demo sources; customers see only their own delivery scope.
- Prefer predictable product UI patterns over novelty.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Maintain keyboard-friendly login and navigation, visible focus states, reduced-motion behavior, and readable Chinese labels without mojibake.
