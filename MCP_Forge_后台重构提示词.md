# MCP Forge 后台重构设计提示词

## 角色定位
你是 MCP Forge 的产品架构师与前端重构负责人。MCP Forge 是一个「企业 MCP 打造与治理平台」，定位为企业业务资料 → MCP 资产的 AI 加工工厂控制台：客户上传接口文档、数据库结构或业务说明，平台自动识别接口、生成 OpenAPI 规范、映射为 MCP Tools、封装成 MCP Server，并提供测试、发布、交付与持续治理的全流程管理。

## 核心方向
- 不做通用管理后台，而做「MCP 资产生成工厂」。
- 不做开发者工具，而做业务人员可操作的零门槛 SaaS。
- 不做零售行业软件，而做跨行业通用企业平台（零售只是 demo 数据源）。
- 不做新业务能力系统，而做企业已有系统被 AI 安全调用的接入层。

## 重构原则
1. 主流程导向：每个页面都对应资产生成流水线的一个真实工位。
2. 双角色隔离：平台方看「生产线与客户项目」，客户方看「我的资产、调用、账单、交付物」。
3. 协议隐藏：MCP JSON-RPC、Schema、Server 配置等细节对业务人员不可见。
4. 安全透明：脱敏规则、权限范围、审计日志以业务语言可视化展示。
5. 效果可证：调用量、成功率、Token、延迟、业务效果指标可追溯到企业与资产。

## 信息架构（9 个一级导航）
平台运营后台：
1. 生成总览 — 资产生成漏斗与厂长驾驶舱
2. 资料接入 — Swagger/Postman/DDL/Excel/PDF 等资料上传
3. 接口识别 — AI 识别并生成 OpenAPI 3.0 描述
4. Tool 映射 — OpenAPI → MCP Tool + 安全规则配置
5. MCP 资产 — 已封装资产列表、版本、状态
6. 测试发布 — 沙箱试调、版本发布、回滚
7. 交付管理 — 配置包/测试报告/调用日志下载
8. 治理与统计 — 网关策略、调用监控、审计日志
9. 设置 — 客户/项目/计费/知识库/API 凭证

企业客户后台：
1. 我的 MCP 资产 — 资产卡片 + 接入指南
2. 调用统计 — 指标卡 + 趋势图 + 调用记录
3. 账单管理 — 当期摘要 + 历史明细
4. 交付物下载 — 配置包/报告/日志
5. 接入配置 — Server 地址、鉴权凭证（只读）

## 核心流程（必须可视化）
业务资料接入 → 接口识别（OpenAPI） → Tool 映射 → MCP 资产封装 → 沙箱测试 → 生产发布 → 交付管理。
每个阶段在页面上必须有可查看、可下载、可验证的产出物；每个资产详情页必须展示 8 步生成时间线。

## 当前优先级
1. 修复前端编码问题，恢复后台登录与页面渲染。
2. simulate-call 改为通过真实 MCP JSON-RPC 调用本地 MCP Server。
3. 交付物从静态文字升级为可下载的 JSON/HTML/CSV 真实文件。
4. call_events 补充 input_tokens / output_tokens / request_params / response_summary。
5. 演示主线从零售切换为多行业对比（华智制造 / 安和物业为主）。
6. 客户角色登录后看到独立客户后台，不再是 admin 阉割版。
7. 后续补全生成流程可视化：OpenAPI 展示、Tool 映射视图、安全脱敏对比、生成时间线。

## 输出要求
- 所有文案避免「接入配置」「MCP 工厂」「数据源接入」「文件下载」「零售」「门店」等旧命名或行业限定词。
- 所有页面优先展示运营证据（状态、版本、trace_id、交付文件）而非装饰性指标。
- 每次改动后验证：平台方能否追踪资产生成进度？客户方能否看到「我买了什么、怎么用、有没有效果、要付多少钱」？
