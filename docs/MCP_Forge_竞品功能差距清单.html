MCP Forge / Competitive Gap Review / 2026-07-13

竞品功能差距清单

这份清单基于《MCP Forge 竞品分析-v2》和当前项目实际代码能力整理，目标不是证明我们“什么都没有”，而是把“竞品强调、市场需要、而我们现在还没真正做实”的能力分层列清楚，方便后续排优先级。

一句话结论
  • MCP Forge 目前的优势在于 AI 候选识别 + 人工审核治理 + 企业后台视角。
  • 最大的缺口不在“页面有没有”，而在 真实运行时、真实认证、真实部署、真实计量。
  • 和竞品相比，你现在更像 企业 MCP 治理与交付中台雏形，还不是完整的 生产级 MCP 运行平台。
本次差距判断依据
  • 竞品文档提到的核心能力：阿里云 MCP、Microsoft MCP Gateway、FastMCP、agentic-community Registry。
  • 当前项目交接文档中的已知缺口：真实 handler、真实文件生成、沙箱仅模拟。
  • 当前代码中的实际实现方式：API Key 本地生成、simulate-call 模拟返回、治理链路已存在。

真实工具调用 / handler

P2 商业化与运营缺口 第三批

不是短板的部分

这几项不要被误判成“我们也没有”。它们恰恰是 MCP Forge 现阶段和竞品拉开定位的地方。

应该继续强化，而不是推翻重做

我们已经有的优势

  • AI 候选识别 + 人工审核链路：这比单纯 OpenAPI 生成器更接近企业真实交付流程。
  • 发布前人工门禁：已经有 open review tasks、manual screen、acceptance checklist 的阻断逻辑。
  • 企业后台视角：管理员与客户视角并存，这一点明显区别于面向开发者的工具型竞品。

正确的产品判断

  • 不要为了追竞品，把审核治理优势做丢。
  • 下一阶段应补 runtime / gateway / auth / billing，而不是先做更大更花的首页。
  • 你的路线更像“企业 MCP 资产交付与运营平台”，不是单纯的 MCP 代码生成器。
