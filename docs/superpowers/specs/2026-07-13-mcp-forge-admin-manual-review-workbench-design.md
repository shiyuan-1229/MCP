# MCP Forge 管理员人工审核工作台设计

## 1. Feature Summary

本次设计面向 `MCP Forge` 管理员端，补齐一张独立的 `人工审核工作台` 原型页，用来处理 AI 识别偏差带来的高风险候选、低置信度结果、敏感字段命中和发布阻断问题。

这不是一个新的通用后台，而是嵌入现有 `资料接入 -> 接口识别 -> Tool 映射 -> MCP 资产 -> 测试发布` 主链路中的风险闸门页面，服务对象是平台管理员、交付负责人、安全复核人。

## 2. Primary User Action

管理员进入页面后，最重要的动作是：

`快速识别高风险候选 -> 查看风险原因与来源冲突 -> 做出人工决策 -> 决定是否允许进入下一阶段`

## 3. Design Direction

- Color strategy: `Restrained`
- Theme scene sentence:
  `一位平台交付负责人在白天办公环境中处理待审核候选，需要在密集信息下快速判断风险，界面应稳定、克制、可信，不制造装饰性干扰。`
- Anchor references:
  - `Linear` 的任务密度与优先级表达
  - `Stripe Dashboard` 的结构清晰和风险提示节奏
  - `Figma Inspect / Dev Mode` 的双栏查看与详情下钻方式

本页延续你当前管理端的深色侧栏、浅色内容区、蓝色主操作色和表格型产品界面，不做品牌方向重置。

## 4. Scope

- Fidelity: `中保真`
- Breadth: `1 个独立页面`
- Interactivity: `静态可浏览原型，局部模拟按钮状态，不接真实接口`
- Time intent: `作为本轮产品与交互参考图，后续可映射开发`

## 5. Layout Strategy

页面采用 `顶部风险看板 + 左侧审核队列 + 右侧详情工作区 + 底部发布阻断与复盘` 的结构。

- 顶部先看全局风险态势，回答“现在哪些最该人工介入”
- 中段左栏承接审核对象列表，强调优先级、状态和责任分发
- 中段右栏承接单条候选详情，回答“为什么拦、拦什么、该怎么改”
- 下段展示发布阻断清单与复盘回流，让人工干预成为闭环，而不是一次性判断

整体节奏以任务效率优先，不使用营销式大卡片和装饰性英雄区。

## 6. Key States

### 6.1 默认态

显示风险指标、待审核队列、当前选中候选详情、人工决策区和发布阻断区。

### 6.2 高风险态

高风险候选在列表中置顶，并使用更强的危险色标签、敏感字段提醒、双人审核标识。

### 6.3 低置信度态

重点显示 `confidence` 和 AI 解释不足，提示管理员补充业务判断。

### 6.4 冲突态

当 DB / OpenAPI / 文档三方信息冲突时，详情区需要并排展示来源差异。

### 6.5 已处理态

显示“已通过 / 已驳回 / 修改后通过 / 已升级复核”的决策结果和处理人。

### 6.6 发布阻断态

明确列出：

- 审核任务未清零
- 人工初筛未通过
- 验收清单未完成

### 6.7 空态

如果暂无待审核候选，应显示“当前没有需要人工介入的高风险候选”，而不是空白表格。

## 7. Interaction Model

- 左侧点击候选，右侧切换详情
- 顶部卡片支持作为筛选入口，如 `只看高风险`、`只看发布阻断`
- 详情区支持查看：
  - AI 识别摘要
  - 风险原因
  - 敏感字段命中
  - 来源冲突
  - AI 版本与人工修订版本对比
- 人工决策区提供四个主动作：
  - `通过`
  - `驳回`
  - `修改后通过`
  - `升级给安全 / 高级审核人`
- 发布阻断区以 checklist 形式呈现
- 复盘区显示高频误判原因，形成“处理一次，沉淀一次”的产品闭环

## 8. Content Requirements

页面需要明确出现以下文案与数据要素：

- 风险 KPI：
  - `待人工审核`
  - `高风险候选`
  - `低置信度`
  - `敏感字段命中`
  - `发布阻断中`
- 队列字段：
  - 项目
  - 候选名称
  - 风险等级
  - 置信度
  - gate reasons
  - 审核要求
- 详情字段：
  - AI 识别摘要
  - 建议业务域
  - 来源资料
  - 敏感字段命中
  - 风险说明
  - 来源冲突对比
  - Tool 修订对比
- 决策字段：
  - 审核结果
  - 修正原因
  - 指派角色
  - 操作日志
- 发布前阻断字段：
  - 审核任务清零状态
  - 人工初筛状态
  - 6 项验收清单
- 复盘字段：
  - 高频误判类型
  - 本周复盘数
  - 需要补强的规则
