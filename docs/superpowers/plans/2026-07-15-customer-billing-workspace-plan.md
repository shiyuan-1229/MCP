# 客户账单工作台 Implementation Plan

**Goal:** 将客户侧“账单管理”从重复摘要展示升级为可理解费用、判断额度并导出对账资料的账单工作台。

**Scope:** 仅改客户侧 HTML、渲染逻辑、样式和回归测试，复用 `/api/platform/billing` 已返回的 `base_amount`、`overage_amount`、`total_amount`、`usage_count`、`period` 与 `status` 字段。

## 页面结构

1. 本期账单：显示应付金额、账单状态、计费周期、到期日和“导出对账单”动作。
2. 费用明细：固定展示订阅服务、超额调用和应付合计；备注关联当前交付 MCP。
3. 调用与额度：按客户套餐提供默认调用额度，显示已用量、剩余额度、百分比和进度条。
4. 历史账单：以表格展示所有账期、金额、状态、调用量和导出动作；无历史记录时显示空状态。

## 实施步骤

- 更新 `mcp/poc/admin/index.html`，替换重复的账单观察卡，增加费用明细、调用额度和历史账单容器。
- 更新 `mcp/poc/admin/assets/modules/renderers.js`，按账期选择当前账单，计算费用构成与套餐额度，渲染历史账单表并复用现有 `exportBillingStatement()`。
- 更新 `mcp/poc/admin/assets/styles.css`，增加账单 Hero、费用行、额度进度条和响应式表格样式。
- 新增 `mcp/poc/admin/tests/customer-billing-workspace.test.mjs`，验证页面容器、渲染函数、费用字段、额度与导出动作。
- 运行客户账单、登录和页面回归测试；以绿城账号验证账单金额为 ¥12,800、调用量为 6,320、账单导出入口可用。

## 验收标准

- 本期账单不再出现重复的“账单观察、确认状态、最近账期”卡片。
- 客户可看到订阅费、超额费、应付合计、调用量和套餐额度。
- 历史账单以可扫描表格展示，单条账单时仍不会造成大面积空白。
- 导出对账单继续复用现有客户端导出能力，不引入新的财务或开票后端流程。
