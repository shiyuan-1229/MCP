# MCP Forge 客户 Demo 方案脚本

> 面向零售客户，展示从「旧系统 MCP 化」到「微信 AI Agent 闭环」的全流程
> 演示时长：约 15-20 分钟

---

## 一、Demo 场景设定

### 场景背景

**客户名称**：华南好邻居连锁便利店（假设客户，覆盖 200+ 门店）

**客户痛点**：
- 有一套老旧的门店管理系统（POS + 库存 + 会员），没 API，数据出不来
- 看到瑞幸发了 MCP、微信发了 AI 专属卡，但不知道怎么接入
- 老板想："顾客能不能直接在微信里问商品库存、查积分、下单？"

**我们的方案**：
通过 MCP Forge，对客户的旧系统做两件事：
1. **旧系统整合**：给 POS/库存/会员系统包一层标准化 API 适配层
2. **MCP 化**：把这些 API 一键转成 MCP Server，供微信 AI Agent 调用

---

## 二、Demo 流程总览（15 分钟）

```
Step 1  │ 痛点重演（2 min）
         │  → 顾客在微信里问店员，店员翻系统查半天
         
Step 2  │ 旧系统整合（3 min）
         │  → MCP Forge 连接客户遗留系统，自动生成标准化 API
         
Step 3  │ API → MCP 一键生成（3 min）
         │  → 在 Forge 界面选中 API，一键生成 MCP Server
         
Step 4  │ 微信 AI Agent 接入（3 min）
         │  → 配置 Claude/微信 Agent，现场演示自然语言交互
         
Step 5  │ 完整闭环演示（2 min）
         │  → 查库存 → AI 推荐 → 下单 → 微信支付
         
Step 6  │ 商业价值总结（2 min）
         │  → 注册费 + 调用费 + 知识库增值，投资回报率预估
```

## 四、Demo 准备清单

### 4.1 技术环境准备

- [ ] 部署 MCP Forge MVP（Swagger 导入 → 生成 → 运行链路过完）
- [ ] 准备一个模拟的零售 API 后端（Mock 好邻居便利店的 POS/库存/会员）
- [ ] 部署 MCP Server 到云端，确保 SSE 地址可访问
- [ ] 配置好 Claude Desktop 或微信 Agent 测试环境
- [ ] 准备好微信 AI 专属卡支付的沙箱环境

### 4.2 演示物料

- [ ] 微信对话演示录屏（备用，防止现场网络出问题）
- [ ] MCP Forge 产品界面截图（高保真原型或开发版）
- [ ] 1 页商业价值汇总单页（打印或投影）
- [ ] 客户 ROI 计算器（Excel 或在线工具）

### 4.3 话术提示

- 避免说技术术语：不要说 "MCP 协议"、"SSE 传输"，而是说 "一键接入 AI"、"系统上 AI"
- 强调 "系统不用换"：客户最怕的是改造现有系统，反复强调零改动
- 用瑞幸做对比："瑞幸花了几百万自己做了 MCP，现在你花几万块就能一样接入 AI"
