# 五个产品方向 · 应用场景详解

> 针对科传的零售/物业/商业地产客户，每个方向用一个真实场景说清楚

---

## A · 纯 MCP 工厂

### 📍 典型场景：独立 SaaS 厂商开放 API

**背景**：某餐饮 SaaS 厂商「餐行天下」做了 5 年点餐系统，服务 3000+ 商家。老板发现客户开始在微信里问"能不能直接查报表"，但团队没精力研究 MCP 协议。

**使用方式**：
- 餐行天下的后端把已有点餐 API 的 Swagger 文档上传到 MCP Forge
- 勾选「今日营收」「订单列表」「菜品销量排行」3 个接口
- 一键生成 MCP Server，复制链接到 Claude Desktop

**结果**：
- 商家老板在微信里说："帮我看看今天卖了多少钱"
- Claude 调取 MCP → 查 API → 返回实时数据
- 全程 10 分钟完成 MCP 化，餐行天下没改一行代码

**适合客户画像**：有 API 文档、有开发能力、预算有限、想先试试水

## B · 工厂 + 知识库

### 📍 典型场景：物业公司搭建 AI 客服

**背景**：某商业地产物业公司「宁骏物业」，管理 15 个写字楼。客服部每天接 200+ 电话，80% 是重复问题（"空调开到几点""车位怎么租""快递放哪"）。

**使用方式**：
- **MCP 工厂部分**：把物业系统的报修 API、投诉 API、缴费 API 转成 MCP Tool
- **知识库部分**：把《业主手册》《写字楼指南》《收费标准》等 PDF 导入知识库，做成 MCP Resources
- **整体接入**：微信 AI Agent 既会调用 API（处理报修），也会检索知识库（回答问题）

**结果**：
- 业主在微信问："帮我报修 A 栋 12 楼灯管坏了"
- AI Agent：调 MCP 报修 API 创建工单 → 回应 "已报修，预计 30 分钟内师傅到场"
- 业主又问："空调供冷到几点？"
- AI Agent：检索知识库 → "工作日 7:00-22:00，周末 8:00-18:30"

**适合客户画像**：有文档资料、有运维数据、希望通过 AI 降低客服成本

## C · 企业 AI 能力套件

### 📍 典型场景：大型零售集团全面 AI 化

**背景**：「华商百货集团」全国 50+ 门店，会员 200 万。用了 ERP、CRM、POS、WMS 四套系统，彼此独立。老板想："能不能让顾客在微信里一句话完成找商品 → 查库存 → 看会员价 → 下单 → 支付的全流程？"

**使用方式**：
- **旧系统整合**：对四套系统做适配器，统一为标准化 API
- **MCP 工厂**：把统一后的 API 一键转为 20+ 个 MCP Tool
- **知识库**：导入商品知识、会员权益、促销规则
- **Agent 中台**：配置专属 AI 导购 Agent，设定对话策略
- **Token 供应**：所有 API 调用走统一 Token 管理

**结果**：
- 完整闭环：找商品 → 查库存 → 查会员价 → 下单 → 微信支付
- 华商不需要换系统、不需要招 AI 团队、不需要改代码
- 科传收取：旧系统整合费（¥15 万）+ 套件年费（¥20 万/年）+ 调用费（按量）

**适合客户画像**：多系统、多业态的大型集团，愿意为"一站式"付高客单价

## D · 行业模板库

### 📍 典型场景：连锁便利店快速接入 AI

**背景**：科传有 80+ 家中小型连锁便利店客户，需求几乎一模一样——商品查询、门店库存、会员积分、附近门店。按照 A 方向每个客户单独做，成本太高。

**使用方式**：
- 科传预装一套「便利店 MCP 标准模板」：
  - 商品模板：商品搜索、商品详情、价格查询、促销活动
  - 库存模板：门店库存查询、库存预警、到货通知
  - 会员模板：会员信息、积分查询、优惠券查询、积分兑换
  - 门店模板：门店列表、营业状态、自提预约
- 新客户签单后，接入模板 + 配置客户自己的数据源 = 1 小时上线

**结果**：
- 「美佳便利店」签约当天完成接入
- 顾客微信对话："附近美佳有卖农夫山泉 5L 装吗？"
- AI 助手：调 MCP 模板 → 查库存 → "天河北店有 20 箱，¥9.9/桶"
- 美佳每月多接 80+ 笔夜间接单，营收 +15%

**适合客户画像**：中小型零售连锁、需求标准化、看重"快"和"简单"

## E · 安全网关平台

### 📍 典型场景：金融机构合规接入 AI

**背景**：「融信集团」（大型商业地产 + 金融业务）想做 AI 助手，但 IT 总监最担心的是安全问题——"把我们的物业系统和会员系统暴露给 AI，出了数据泄露谁负责？"

**使用方式**：
- 融信的 API 不直接暴露给 AI Agent，而是全部经过 **Forge Shield 安全网关**
- 网关层做四件事：
  1. **身份认证**：只有授权的 AI Agent 才能调用
  2. **权限控制**：AI 只能查自己的会员数据，不能查别人的
  3. **数据脱敏**：返回的会员手机号自动打码（138****8888）
  4. **审计日志**：每次调用都被记录——"某日某时，AI 查询了会员 138****8888 的资料"
- 月末自动生成调用报表：哪个部门调了哪些接口、花了多少钱

**结果**：
- IT 总监放心了：所有 AI 调用都可追溯、可控、可审计
- 融信集团的 CTO："终于找到一个能让我们放心上 AI 的方案"
- 科传收取：安全版月费（¥1.5 万/月）+ 调用费

**适合客户画像**：金融、地产、政府等合规敏感的大型客户，IT 决策链长
