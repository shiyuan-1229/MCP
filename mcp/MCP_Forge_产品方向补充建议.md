# MCP Forge 产品方向补充建议

> 在原有选项 A（纯 MCP 工厂）、B（工厂+知识库）、C（企业 AI 能力套件）基础上，补充两个新方向

---

## 方向 D：垂直场景 MCP 模板库 —— 行业加速器

### 一句话

> **不卖工具，卖「开箱即用的行业 MCP 模板」—— 针对零售/物业等场景预装好业务模块，客户拿来即用。**

### 核心思路

与其让每个客户从零开始封装自己的 API，不如由科传基于行业经验，预先打造一套**标准化的零售业务 MCP 模板库**：

```
┌─────────────────────────────────────────────────────┐
│               MCP 行业模板库（零售版）                    │
├──────────────┬──────────────┬───────────────────────┤
│   📦 商品    │   👤 会员    │   📋 订单              │
│  · 商品查询   │  · 会员信息  │  · 创建订单             │
│  · 库存查询   │  · 积分查询  │  · 订单状态查询         │
│  · 价格查询   │  · 等级查询  │  · 退换货申请           │
│  · 促销查询   │  · 卡券核销  │  · 物流查询             │
├──────────────┼──────────────┼───────────────────────┤
│   🏪 门店    │   📊 数据    │   🤖 智能               │
│  · 门店列表   │  · 销售报表  │  · 商品推荐             │
│  · 营业状态   │  · 库存预警  │  · 智能客服             │
│  · 排队查询   │  · 客流统计  │  · 营销策略推荐         │
│  · 自提预约   │  · 热力图    │  · 异常预警             │
└──────────────┴──────────────┴───────────────────────┘
```

**核心差异**：客户不需要理解 API、Swagger、MCP 协议，只需要说"我要接入微信 AI"，科传直接给一套配好的行业模板。

### 价值点

| 角色 | 收益 |
|------|------|
| **客户** | 零配置、零学习成本、1 小时上线微信 AI 助手 |
| **科传** | 可复制、可规模化、边际交付成本趋近于零 |
| **渠道** | 区域代理商可以拿着模板直接签单，无需技术团队 |

### 适用场景

- 中小型零售连锁（便利店、服装店、生鲜超市）—— 行业痛点高度同质化
- 购物中心/商业地产 —— 商户入驻后需要统一接入会员和营销系统
- 物业公司 —— 报修、缴费、访客等场景标准化程度高

### 盈利模式

| 收费项 | 模式 | 预估价格 |
|-------|------|---------|
| 行业模板授权 | 年费 | ¥20k-¥60k/年 |
| 定制化接入（差异部分） | 项目制 | ¥10k-¥30k |
| 模板市场抽佣（如有三方开发者） | 按收入 | 15-30% |

## 方向 E：MCP Server 托管与安全网关 —— 企业管控平台

### 一句话

> **不做 MCP 生成器，做企业 MCP 的「生产运维平台」—— 生成、部署、监控、安全、审计一条龙。**

### 核心思路

当企业把自己的业务 API 暴露给 AI Agent 时，最担心的不是"能不能生成"，而是**安全问题**。这个方向的定位就是做 MCP 时代的「API 网关 + 运维平台」：

```
                    ┌─────────────────────────────┐
                    │    微信 Agent / Claude / Cursor   │
                    └──────────────┬──────────────┘
                                   │ 调用
                    ┌──────────────▼──────────────┐
                    │    MCP 安全网关（Forge Shield）    │
                    │                                │
                    │  ① 身份认证   ② 权限校验   ③ 限流     │
                    │  ④ 审计日志   ⑤ 敏感数据脱敏  ⑥ 计费    │
                    └──────┬──────┬──────┬──────┬──┘
                           │      │      │      │
                    ┌──────▼─┐ ┌──▼───┐ ┌──▼───┐ ┌──▼───┐
                    │ 商品API │ │会员API│ │订单API│ │更多.. │
                    └────────┘ └──────┘ └──────┘ └──────┘
```

**核心差异**：不只是"生成 MCP Server"，而是让企业**放心把 MCP Server 放到生产环境**。

### 核心功能

| 模块 | 说明 | 价值 |
|------|------|------|
| 🔐 **访问控制** | API Key / OAuth2 / IP 白名单 / 角色权限 | 防止 AI Agent 越权调用 |
| 📝 **审计日志** | 每次调用记录：谁调了、调了什么、传了什么参数 | 满足合规审计要求 |
| 🚦 **限流与熔断** | 按调用方/接口/时间窗口限流，防止被刷 | 保护后端系统稳定 |
| 🔒 **数据脱敏** | 自动识别并脱敏手机号、身份证、银行卡等敏感字段 | 保护客户隐私 |
| 📊 **调用大盘** | 实时监控：调用量、成功率、平均延迟 | 运维可视化 |
| 💳 **计费结算** | 按调用量/接口/客户自动计费，生成账单 | 内部成本分摊 / 对外收费 |

### 价值点

| 角色 | 收益 |
|------|------|
| **企业 IT 负责人** | 不用担心 API 暴露给 AI 后的安全问题，有完整的审计和权限管控 |
| **运维团队** | 统一监控所有 MCP 接口的调用情况，出问题了知道谁在调 |
| **业务方** | 可以按 API 维度核算成本，内部结算透明 |
| **科传** | 从"项目制交付"升级为"平台制持续服务"，续费率大幅提升 |

### 适用场景

- 大型零售集团 —— 多业态、多云、多系统，需要一个统一的管理入口
- 金融/地产客户 —— 合规要求高，不能裸 API 暴露给 AI
- 已经有 API 网关的企业 —— 可以无缝对接现有基础设施

### 盈利模式

| 收费项 | 模式 | 预估价格 |
|-------|------|---------|
| 托管基础版 | 月费 | ¥3k-¥8k/月 |
| 企业安全版（含审计+脱敏） | 月费 | ¥8k-¥20k/月 |
| 私有化部署 | 项目制 | ¥100k-¥300k |
| 调用量阶梯抽成 | 按量 | ¥0.5-¥2/万次 |

## 下周会议讨论建议

如果时间充裕，建议会议这样排序讨论：

1. **首先对齐方向 C（能力套件）**：这是长期愿景，大家先认不认同这个方向？
2. **然后讨论方向 D vs E 作为短期突破口**：
   - 如果客户以中小型为主，且需要快速签单 → 优先 D（行业模板）
   - 如果客户以大中型为主，且安全诉求强烈 → 优先 E（安全网关）
   - 最理想的情况：**D 做获客抓手，E 做增值升级**，两条线并行

> **最终建议组合拳**：方向 C 是北极星，方向 D 打头阵快速获客，方向 E 做差异化壁垒长期升级。
