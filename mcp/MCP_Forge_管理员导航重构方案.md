# MCP Forge 管理员导航重构方案

> 视角：管理员视角
> 产品模式：企业业务资料 -> MCP 资产
> 基准文档：`D:/桌面/mcp方案/mcp/MCP_Forge_agent.md`

---

## 一句话定位

管理员后台不是“客户接入配置台”，而是一个把客户提交的业务接口、业务文档、数据库结构和字段说明，持续加工为 MCP 资产的生成工厂控制台。

---

## 方案目标

本次导航重构的目标，不是简单改名字，而是把管理员对产品的理解路径从“平台运维/接入管理”切换成“资料加工/资产生产/测试交付”。

重构后的管理员后台，需要让管理者一眼看懂三件事：

1. 现在收到了哪些业务资料
2. 这些资料已经被加工到了哪一步
3. 哪些 MCP 资产已经可以测试、发布、交付和治理

---

## 导航结构

推荐管理员一级导航顺序如下：

1. 生成总览
2. 资料接入
3. 接口识别
4. Tool 映射
5. MCP 资产
6. 测试发布
7. 交付管理
8. 治理与统计
9. 设置

这套顺序的核心原则是：

- 前 7 项是主流程导航
- 第 8 项是横向运营治理
- 第 9 项是底层配置和支撑能力

---

## 页面职责

### 1. 生成总览

定位：管理员首页，负责展示整条“企业业务资料 -> MCP 资产”生产线的状态。

应展示：

- 本周新增业务资料
- 待识别资料数
- 已生成 OpenAPI 数
- 已映射 Tools 数
- 已封装 MCP Server 数
- 待测试 / 待发布 / 待交付资产数
- 最近生成动态
- 主流程看板

### 2. 资料接入

定位：主流程第一入口，负责登记、接收和归档客户业务资料。

应展示：

- 客户名称 / 项目名称
- 资料类型：Swagger、Postman、SQL/DDL、字段说明、示例数据、业务文档
- 导入状态
- 资料完整度
- 最近更新时间
- 下一步动作：提交识别

### 3. 接口识别

定位：主流程第二步，负责让 AI 把业务资料整理成 OpenAPI 草稿。

应展示：

- 关联资料
- 识别出的接口数
- 参数与返回结构
- 字段疑点 / 待确认项
- OpenAPI 草稿状态
- 下一步动作：进入 Tool 映射

### 4. Tool 映射

定位：主流程第三步，负责把 OpenAPI 端点转成 MCP Tools。

应展示：

- 接口与 Tool 的映射关系
- Tool 名称、描述、参数 schema
- 安全规则：鉴权、脱敏、限流
- 映射完整度
- 下一步动作：封装为 MCP 资产

### 5. MCP 资产

定位：主流程第四步，负责展示已经封装完成或封装中的 MCP Server 资产。

应展示：

- 资产名称
- 所属客户 / 项目
- Tools 数量
- 资产版本
- 封装状态
- 生成时间线
- 下一步动作：进入测试发布

### 6. 测试发布

定位：主流程第五步，负责沙箱验证、版本管理和发版。

应展示：

- 资产版本列表
- 沙箱试调
- 测试结果
- 灰度 / 正式发布状态
- 回滚能力

### 7. 交付管理

定位：主流程第六步，负责交付物打包、下载和交付确认。

应展示：

- 交付包
- 测试报告
- 接口说明
- 运行说明
- 下载状态
- 交付确认状态

### 8. 治理与统计

定位：上线后的横向治理中心，负责观察资产运行质量。

应展示：

- 调用量
- 成功率
- Top Tools
- 运行异常
- Webhook 日志
- 健康检查
- 审计历史

### 9. 设置

定位：非主流程支撑页，负责底层管理配置。

应归入：

- 客户管理
- 项目配置
- 凭证与协议
- 计费管理
- 知识库配置

## 主流程路径

管理员在后台中的正确理解路径应该是：

`资料接入 -> 接口识别 -> Tool 映射 -> MCP 资产 -> 测试发布 -> 交付管理`

横向辅助路径：

- `治理与统计`：用于资产上线后的持续监控
- `设置`：用于系统级支撑配置

这意味着：

- `接入配置` 不再作为第一入口
- `知识库` 不再作为主流程页面
- `网关与治理` 不再作为过早出现的一线主导航

---

## 管理员视角导航原则

### 原则 1：导航命名必须是生产动作，不是技术名词堆叠

优先使用：

- 资料接入
- 接口识别
- Tool 映射
- MCP 资产
- 测试发布
- 交付管理

避免使用：

- 接入配置
- 客户接入
- 文件下载
- 平台工厂

### 原则 2：先展示“资料如何变资产”，再展示“资产如何被治理”

如果管理员打开后台第一眼看到的是证书、Webhook、OAuth、SSE，会误以为产品是一个运维平台。

如果第一眼看到的是：

- 新增资料
- OpenAPI 草稿
- Tool 映射状态
- MCP 资产产出

那么产品叙事就会稳定在“资产生成工厂”。

### 原则 3：客户行业词不能抢走平台主叙事

零售、门店、销售、校园等词只能存在于项目内容层，不能存在于一级导航层。

一级导航必须永远表达平台能力，而不是某个 demo 行业。

## 原型说明

本次原型图重点展示的是“管理员视角导航栏”和其对应的首页信息布局，因此原型会强调：

- 左侧纵向一级导航
- 主流程步骤感
- 当前页高亮
- 生成总览的工厂看板结构
- 页面之间的上下游关系

原型不是视觉营销页，而是任务型后台页面。
