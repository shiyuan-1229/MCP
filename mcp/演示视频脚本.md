# MCP Forge 演示视频脚本

## 视频标题
MCP Forge - 中小零售商业务能力标准化为 MCP 接口的平台

## 视频时长
3-5 分钟

## 视频结构

### 1. 开场 (30秒)
- 屏幕显示：MCP Forge 标题 + 副标题
- 语音："大家好，我是董嘉豪，今天我将展示 MCP Forge 项目。这是一个将中小零售商业务能力标准化为 MCP 接口的平台。"

### 2. 问题背景 (30秒)
- 屏幕显示：中小零售商面临的挑战
- 语音："传统零售商在接入微信 AI 时面临三大挑战：技术门槛高、开发成本大、缺乏标准化方案。MCP Forge 旨在解决这些问题。"

### 3. 核心功能演示 (2分钟)
- **步骤1：创建订单**
  - 屏幕显示：微信对话界面
  - 语音："顾客在微信中输入：'帮我下个单：元气森林×2、农夫山泉×1，送美佳门店自提。'"
  - 屏幕显示：AI 自动调用 create_order Tool
  - 屏幕显示：订单创建成功，返回订单号、总价、取货码

- **步骤2：支付流程**
  - 屏幕显示：订单状态为"待支付"
  - 语音："AI 调用 process_payment Tool，生成微信支付链接"
  - 屏幕显示：支付链接和支付状态
  - 语音："顾客点击链接，直接跳转到微信支付页面"

- **步骤3：管理后台**
  - 屏幕显示：管理后台登录页面
  - 语音："商家可以通过管理后台查看订单状态和支付情况"
  - 屏幕显示：订单管理页面，显示支付状态为"支付中"

### 4. 技术亮点 (30秒)
- 屏幕显示：技术栈和架构图
- 语音："MCP Forge 使用 Node.js + Express + SQLite，通过 MCP 协议实现与微信 AI 的无缝对接。支付功能支持点击按钮支付，提升用户体验。"

### 5. 商业价值 (30秒)
- 屏幕显示：目标客户和商业模式
- 语音："我们的目标客户是中小型零售连锁，商业模式是注册费 + 调用费。MCP Forge 让零售商像点外卖一样接入微信 AI。"

### 6. 结尾 (30秒)
- 屏幕显示：项目成果和联系方式
- 语音："MCP Forge 已完成 POC 验证，所有功能都已实现。感谢观看，欢迎大家交流。"

## 演示要点

1. **完整购物流程**：从下单到支付的全流程演示
2. **支付功能**：重点展示点击按钮支付的便捷性
3. **管理后台**：展示商家视角的订单管理
4. **技术实现**：简要说明技术栈和架构
5. **商业价值**：强调目标客户和商业模式

## 视频拍摄建议

- 使用屏幕录制软件（如 OBS）
- 准备好微信 AI 智能体链接
- 确保服务器正常运行
- 录制前测试所有功能
- 保持语速适中，重点突出

## 预期效果

通过这个视频，观众可以清晰了解：
- MCP Forge 的核心功能和价值
- 如何帮助中小零售商接入微信 AI
- 支付功能的实现和优势
- 项目的完整性和实用性
