## 方案要点 / v2 更新

- 【新增背景】项目发起方为**科传**（智慧零售引领者），核心定位是帮助零售客户将旧系统（ERP/CRM/POS等）MCP 化，通过微信 Agent 抢占新流量入口。
- 【新增服务】除 MCP 转换外，提供知识库搭建、旧系统集成等增值盈利服务。
- 【新增计费】补充了**按注册计费**与**按调用计费**两种模式及组合策略建议。
- 【新增会议准备】针对下周产品方向与盈利模式讨论会议，整理了议题框架、通晒清单、需要会前确认的关键问题。
- 【战略升级】将"用套件思维卖大单，用工具思维建生态"升级为：**MCP 工厂做抓手，整旧系统 + 建知识库，注册+调用双计费，抢占微信 AI 生态红利**。

- 产品定位：企业 API 的 AI 连接器，不单卖工具，打包 Token + Agent + 知识库 + MCP 形成企业 AI 能力套件。
- 技术架构：输入层 → 协议转换引擎 → 生成与运行层 → 验证与分发层。
- MVP 范围：Swagger 导入、API 选择、Node.js MCP Server 生成、云端托管、Claude Desktop 验证。
- 商业化：Freemium 开发者工具 → 企业年费套件 → MCP 服务市场抽佣。
- 协同机会：与微信 AI 专属卡/AI 支付结合，打造"查库存 → 推荐 → 下单 → 微信支付"闭环 demo。
- 风险提示：协议迭代、安全越权、巨头自研、客户认知、生成质量。
